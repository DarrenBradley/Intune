Freigabe der INDM-App

Der Client für den INDM kann über den Microsoft Store for Business freigegeben werden und via Intune und SCEM automatisch verteilt werden. Der Link zum INDM im Microsoft Store for Business lautet:

https://businessstore.microsoft.com/de-de/store/details/INDM/9NWVZR414XS6

Wie der Microsoft Store for Business in Intune oder SCEM eingebunden wird, kann hier nachgelesen werden:

Einrichten des Autostarts für INDM

INDM richtet sich als Startup-Task ein und wird aktiviert, wenn der Nutzer das erste mal auf das Appicon im Startmenü klickt. Um eine bessere Nutzererfahrung zu gewähren, kann der Autostart von INDM auch über Microsoft Intune automatisch aktiviert werden.

Um den Autostart zu aktivieren muss ein Windows 10 Device Restriction profile erstellt werden. Im Device Restriction profile muss der Package Family Name HaukeGtze.IntuneNetworkDriveMapping_6bk20wvc8rfx2 im Punkt App Store > Startup apps hinzugefügt werden.

Abbildung 1: Der Package Family Name wurde in Startup apps eingetragen.

Installation des ADMX-Templates

Um INDM konfigurieren zu können, muss das ADMX-Template für INDM über ADMX-Ingestion installiert werden:

1.       Erstellen Sie dazu in Microsoft Intune ein benutzerdefinitieres Windows 10 profile.

2.       Fügen Sie mit Add einen neuen OMA-URI Eintrag hinzu.

3.       Tragen Sie als Name Intune Network Drive Mapper ADMX-Ingestion ein.

4.       Geben Sie als OMA-URI den Wert ./Vendor/MSFT/Policy/ConfigOperations/ADMXInstall/weatherlights/Policy/IntuneNetworkDriveMapper ein.

5.       Geben Sie als Wert den folgenden Inhalt ein:

INDM konfigurieren

Um INDM zu konfigurieren kann das im vorherigen Schritt angelegte Windows 10 benutzerdefinierte Profile verwendet werden, oder ein neues benutzerdefiniertes Windows 10 profile angelegt werden. Die folgenden OMA-URIs lassen sich konfigurieren:

OMA-URI

Beschreibung

 

Beispielwert (String)

./User/Vendor/MSFT/Policy/Config/weatherlights~Policy~IntuneNetworkShareMapper~Configuration/Enabled

Aktiviert INDM. Standardmäßig: disabled

<enabled/> oder <disabled/>

./User/Vendor/MSFT/Policy/Config/weatherlights~Policy~IntuneNetworkShareMapper~Configuration/RefreshInterval

Legt das Aktualisierungsinterval für Laufwerke in ms fest.

<enabled/><data id="RefreshInterval" value="9000"/>

./User/Vendor/MSFT/Policy/Config/weatherlightscom~Policy~IntuneNetworkShareMapper~Configuration/MapPersistent

Verbindet ein Laufwerk als persistent. Kann genutzt werden, wenn INDM nicht dauerhaft laufen soll.

Standardmäßig: disabled

<enabled/> oder <disabled/>

./User/Vendor/MSFT/Policy/Config/weatherlights~Policy~IntuneNetworkShareMapper~Policy/NetworkDriveMappings

Mit dieser Einstellung können zu verbindenen Laufwerke übermittelt werden. Ein Eintrag in der Liste folgt folgendem Schema:

<NUMMER DES EINTRAG>&#xF000;<LAUFWERKBUCHSTABE>;<NETZWERKPFAD>&#xF000;

Mit &#xF000; wird eine weitere Zeile hinzugefügt.

<enabled/><data id="NetworkDriveMappingsList" value="1&#xF000;P; \\server1.domain.local\share2&#xF000;2&#xF000;F;\\server1.domain.local\share1"/>

Funktionsweise

·       Der INDM überwacht im festgelegten Aktualisierungsinterval die Netzwerkverbindung auf mögliche Änderungen.

·       Wird eine Änderung der Netzwerkkonfiguration erkannt, versucht der INDM 15 mal eine Verbindung mit den konfigurierten Laufwerken herzustellen.

·       Wurde ein Laufwerk bereits verbunden wird geprüft, ob der eingebundene Netzwerkpfad auch dem konfigurierten Netzwerkpfad entspricht.

o   Trifft dies zu, ignoriert INDM das Laufwerk.

o   Trifft dies nicht zu, wird das Laufwerk mit dem konfigurierten Pfad neu verbunden.

·       Sind die 15 Verbindungsversuche verbraucht, wartet INDM wieder auf eine Änderung der Netzwerkverbindung.

Troubleshooting

Laufwerke werden nicht verbunden.

·       Ursache 1: Der Laufwerkbuchstabe wird möglicherweise schon verwendet. Es sollte ggf. USB-Laufwerke usw. geprüft werden.

·       Ursache 2: Der konfigurierte Netzwerkpfad ist falsch.

·       Ursache 3: Es liegt ein Berechtigungsproblem für den Netzwerkpfad vor. Es sollte versucht werden den Netzwerkpfad manuell in die Adresszeile einzugeben und überprüft werden, ob eine Anmeldeaufforderung oder Fehlermeldung angezeigt wird.

Wie kann ich sehen, ob meine Konfiguration richtig übertragen wurde?

Prüfe den Registrierungsschlüssel: HKCU\Software\Policies\weatherlights.com
